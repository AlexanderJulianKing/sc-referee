sc-referee · decision logic

The if-then machine

sc-referee is two stages. First it identifies what the analysis is and what every column means. Then it applies the specific statistical checks that fit — and each one recomputes the correct arithmetic to decide whether a rule was broken.

Claude proposes→ a human confirms once→ arithmetic decides

◆The pipeline at a glance

Data comes in on the left; a verdict comes out on the right. Nothing can fail CI until a human ratifies the design (the amber gate). Everything after the gate is deterministic and re-runs identically forever.

▤The six verdicts

Only blocker fails CI. Everything else is posted as an annotation — never a silent green, never a false accusation.

blockeryour science is wrong — CI fails
majorlikely wrong; posted, not gated needs_evidencewe couldn't be sure — abstain
informationalmeasured fact, not a defect not_auditedwe didn't look (not "clean") passrecomputed and it holds up

⧉Multi-file — one analysis, many files

Real analyses are often sharded: 8 mice in 8 .h5ad files, or two treatments in two CSVs. sc-referee audits the assembled analysis — and refuses to silently audit one shard as if it were the whole study.

▶ verified, not trustedre-derive what it can; refuse the rest — the assembler fails closed fails closed

A manifest is a proposal. It declares only the semantically-undecidable part (which file means what); everything checkable the assembler recomputes, and on anything that would silently corrupt or narrow the scope it refuses rather than proceed.

  • a shard is normalized / mixed / has duplicate genes or cell-ids→refuse
  • the assembled sample set ≠ the declared expected, or a matrix on disk is unlisted→refuse — never a partial scope
  • a constant would overwrite real metadata; a shard is a non-RNA modality; a matrix is transposed→refuse
  • the manifest content or any shard/obs file changed after confirmation→refuse — digest + per-file hashes
  • the manifest is unconfirmed→ every earned blocker is downgraded, and a clean run is neutral — never a green pass on an unratified layout
The one thing it can't verify is the one thing you confirm. Arithmetic re-derives counts-ness, gene alignment, and cell-id uniqueness — but it cannot know that M1.h5ad is really the WT mouse. That file→label map is exactly what the human ratifies; a pass means "given your confirmed map, the statistics hold," never "your labels are right." The confirm screen shows the whole scope (included shards + excluded files) so that judgment is reviewable.

1Identify — get the ducks in a row

Click any step for the exact rule.

▶ ingestread the matrix, label what it is
  • an .h5ad under the folder→ read it (AnnData adapter)
  • no .h5ad, but counts.csv/.tsv + an obs table→ read those (CSV adapter) — you don't need the scanpy stack to be audited
  • a sc-referee.manifest.yaml is present→ assemble the declared shards into one matrix (§ Multi-file)
  • >1 candidate matrix and no manifest→ refuse — never guess which is the analysis (an atlas.h5ad decoy, one arm of a split, one mouse of eight)
  • neither is present→ error (nothing to read)
  • matrix read→ label the measure: integer-valued ⇒ counts; otherwise normalized — recorded, not a hard refusal
  • either way→ build the bundle: detect a replicate column by name, bind a reported-results table if its header looks like DE output, and parse (never execute) the analysis code into signals
Normalized data no longer fails at the door. An earlier build refused the whole run on non-integer values — too blunt: some checks (confounding, double-dipping) never touch the matrix. Now the measure is labeled, and only the checks that need raw counts abstain (not_audited: "the recompute needs raw counts; this matrix is normalized — NOT checked"). The structural checks still run.
▶ classifywhat is this, and what does each column mean?
  • deterministic first: exactly one replicate-named column (cardinality 2–50) AND one condition-named column (2–6)→ propose it, Claude is never called
  • a role is ambiguous (0 or >1 candidate) AND an API key is present→ Claude fills roles only via a forced tool call (it cannot author a formula — the schema won't let it)
  • ambiguous and no model available→ a heuristic draft, everything stamped low-confidence
The model proposes roles; arithmetic writes the formula. Claude assigns meaning — "group is the condition, donor is the replicate." Deterministic code then synthesizes the model formula, the contrasts, and the coefficient from those roles and the data. A language model never authors executable statistics.
▶ confirmthe one human gate
  • proposal written→ always confirmed_by_human: false
  • a person runs sc-referee confirm→ the flag flips to true
  • still unconfirmed at audit time→ every earned blocker is downgraded to needs_evidence

2Apply — did the analysis break a rule?

Each check declares which analysis types it applies to and the worst verdict it may emit (its block entitlement). Click a check for its full if-then ladder and real thresholds.

▶ confoundingis the condition effect even estimable? can block
applies to condition_contrast_DEreads only .obs power-independent — the guaranteed blocker

Exact design-matrix algebra. The one place the tool draws a hard mathematical line: if the condition is perfectly aliased with a batch, no model can separate them.

  • a covariate varies within a sample, or a declared level is missing→needs_evidence (config, not science)
  • R²(condition ~ nuisance) == 1 (aliased; the only pure-math threshold)→blocker — re-run the experiment
  • a declared batch is omitted and its partial R² ≥ 0.01→major — add the term
  • estimable but near-collinear, VIF ≥ 10→informational — efficiency cost, not a confound
  • otherwise→pass
The blocker is doubly gated: it fires only when the design is confirmed AND the condition role is high-confidence. If Claude was unsure which column is the condition, an aliasing result is capped at needs_evidence — the tool never blocks on a role it isn't sure it read correctly.
The decision statistic is partial R², not per-term leakage λ: λ shrinks mechanically as a batch gains levels, so a 40-run batch 97% aliased with condition scored a deceptively-tiny λ and slipped the old cut. Partial R² is cardinality-invariant — it was measuring the confound, not the batch's number of levels. Caught in adversarial review.
▶ experimental_unitwere cells treated as replicates? (pseudoreplication) can block
applies to condition_contrast_DE + a recorded replicate + unit_of_test = cell blocker on pydeseq2, capped at major on simple

Recompute the differential expression at the replicate (donor) level — pseudobulk NB — and see how many of the claimed genes survive. Every abstain-gate is checked before survival, so an underpowered recompute can never masquerade as a blocker.

  • matrix normalized, unit unresolved, or no replicate recorded — the recompute can't run→not_audited — "NOT checked", never a silent pass
  • ran, but not comparable, 0 claimed genes testable, or covariates vary→needs_evidence
  • fewer than 3 biological replicates per arm→needs_evidence
  • not powered — <80% of claimed genes individually detectable (MDE ≤ |log2FC| 1)→needs_evidence — exploratory, collapse is ambiguous
  • powered AND survival ≤ 0.10→blocker — claims collapse at the right unit
  • survival < 0.60→major — only partly survive
  • survival ≥ 0.60→pass — holds up at the replicate level
Blocks when powered, abstains when not. On 8 well-powered donors a pseudoreplicated analysis earns an honest blocker; on 4 it correctly returns needs_evidence. Validated against muscat's planted truth + Squair 2021 across 120 simulated datasets. A replicate counts as recorded whenever your confirmed design names a column that exists in the data — the design is authoritative, so a donor column named replicate can't slip past a token list.
▶ multiple_testingwas the p-value list actually corrected? can block
applies to condition_contrast_DE with a reported table reads only the table — no data, no code, no model

Recompute Benjamini–Hochberg over the analyst's own p-values. The cheapest exact check, and the most common error class in the expert rubrics.

  • nothing claimed significant→pass
  • a legitimate correction was applied and ≥60% survive→pass
  • a valid less-conservative method (Storey q ≤ BH)→informational — never accused
  • uncorrected but the omission changed nothing (≥60% survive)→informational — apply BH anyway
  • uncorrected / impossible adjustment, confirmed, and ≤10% survive BH→blocker — overwhelmingly false positives
  • uncorrected, 10–60% survive→major
▶ count_modelright unit, wrong model (a t-test on counts) max: major
applies to condition_contrast_DE + unit_of_test = sample + counts never a blocker — a wrong model is "suboptimal but acceptable"

The measured frontier failure: a strong model uses the right unit (pseudobulk) but then runs a t-test / OLS on log-CPM instead of a count model. Recompute with NB and report what it missed.

  • the matrix is normalized (no raw counts to re-fit an NB against)→not_audited — "model NOT checked"
  • no code to read (NB vs t-test is invisible from the table alone)→needs_evidence
  • both a count method and a non-count test appear→needs_evidence — can't tell which made the table
  • a count model (DESeq2/edgeR/voom…) was used→pass
  • a non-count test on counts (t-test, Mann–Whitney, OLS)→major — counts aren't Gaussian; here's what NB finds
▶ double_dippingclustered, then tested the same cells (circular analysis) flags for review
applies to marker_detection + unit_of_test = cell + a marker test whose grouping is data-derived (by provenance, not method name) structural — power-independent, like confounding

Cluster cells from the expression matrix, then test for markers between those clusters on the same cells: the groups were chosen to separate the cells, so the test is anti-conservative — it "finds" markers even in pure noise.

  • only descriptive rankings, no calibrated p-values claimed→informational
  • a selection-aware safeguard keyword is in the code (count-splitting / held-out set / ClusterDE)→needs_evidence — a keyword isn't proof it's applied correctly; verify the contract
  • de-novo clustering feeding a marker test, p-values claimed, no safeguard→needs_evidence — the double-dipping structure
The claim is about calibration, not truth: "the reported p-values are not valid for post-clustering inference" — never "the markers are false." Method-agnostic: a grouping counts as de-novo when its values trace back to the expression matrix through the code (a bespoke GaussianMixture(...).fit_predict(X_pca) is caught exactly like leiden), and a provably-predefined grouping vetoes an incidental clustering call (a leiden for a UMAP + a marker test on genotype is not flagged). Phase A's provenance is may-level, so the check escalates rather than accuses — the strongest verdict is needs_evidence; a hard blocker awaits the deferred must/overlap analysis, so a syntactic-but-not-semantic read of X can never false-accuse.
▶ effect_size_thresholdsignificant — but is the effect big enough to mean anything? max: major
applies to condition_contrast_DE + a reported table with log2FC + significance never a blocker — a small-but-real effect is still real

With thousands of cells the p-value collapses to zero on effects too tiny to matter: a gene at |log2FC| < 0.25 (under a ~19% change) can be "significant" and biologically meaningless. When the significant list is dominated by such genes, significance is being driven by n, not effect.

  • no log2FC column, or no p/padj column to find the claimed discoveries→needs_evidence
  • nothing was claimed significant→pass — no over-claim to make
  • ≥ 50% of significant genes are negligible (|log2FC| < 0.25)→major — significance ≠ discovery here
  • ≥ 10% negligible→informational — worth noting, not a defect
  • otherwise→pass
Thresholds quoted from the code: NEGLIGIBLE = 0.25, MAJOR_FRACTION = 0.50, INFO_FRACTION = 0.10. This flags the interpretation, not the arithmetic — the p-values may be perfectly correct; the claim that a 15% change is a "finding" is what breaks. That's an analyst's call, so it caps at major.

3Decide — one verdict for the run

The findings collapse to a single conclusion, with two hard rules protecting specificity.

  • worst status wins→ the report's headline is the most severe finding
  • only blocker fails CI→ major / needs_evidence / not_audited are posted as neutral annotations
  • nothing ran for a recognized type→not_audited — a green run then means "we didn't look," not "clean"
  • a check tries to exceed its entitlement→ the spine clamps it down — a verifier can't manufacture a blocker
Config is not science. A level or column that doesn't exist in the data raises a configuration error and exits separately — it is never dressed up as a blocker. "Blocker" always means your science is wrong, never your YAML is wrong.

4Fix — the actionable other half

A linter that only flags is half a tool. For every flagged check, sc-referee fix emits the correction — generated from the confirmed design, never from an LLM.

  • pseudoreplication (experimental_unit)→ a runnable pseudobulk script: sum raw counts to one sample per (donor × condition), fit DESeq2 at the replicate level, write the corrected gene list
  • confounding→ aliased (R²=1) → "re-run the experiment, cross the batch with condition"; an omitted batch → the exact term to add to the model
  • uncorrected testing→ the three lines of Benjamini–Hochberg over the full tested family
  • wrong model (t-test on counts)→ swap in a count model (DESeq2 / edgeR) on the raw counts
  • negligible effects→ add a fold-change floor (abs(log2FC) ≥ 1.0) or test against a threshold directly (edgeR glmTreat)
  • double-dipping→ a selection-aware recipe: count-split, held-out clustering, or ClusterDE
Only blocker and major findings get a fix — a pass, an abstention, or an advisory has nothing to correct. The pseudobulk script is built from your replicate and condition roles (not the raw model string), so it always runs against the columns it constructs. The verdict tells you what broke; the fix hands you the reanalysis.

⛓Bundle — a whole multi-step analysis

A real Claude-Science-style analysis isn't one clean result — it's a chain of numbered scripts, data files, and a report full of confident claims. sc-referee bundle <folder|.zip> parses that pipeline, never runs it, inventories the steps, and says honestly what it can audit. It renders no pass and no blocker: a bundle carries no human ratification in place, so its strongest verdict is needs_evidence.

▶ coverage verdictwhat can this pipeline even be checked for? honest, not asserted
  • ≥1 step runs a differential-expression analysis a check knows→auditable — confirm a design to audit them
  • no step runs an analysis sc-referee checks (e.g. a splicing / PSI pipeline)→not_audited — "green here would mean 'not looked at', not 'clean'"
The specificity rule applied to a whole pipeline: it never calls a bundle clean just because it ran without error.
▶ provenanceis a marker test's grouping derived from the data it tests? flags for review

Double-dipping runs over the bundle by data-flow, not method name: it traces each marker test's groupby column back through the code. If the column's values came from the expression matrix — a cluster label, an annotated cluster, a bespoke GaussianMixture(...).fit_predict(X_pca) — the test is circular; if they trace only to predefined metadata (genotype), it isn't. Taint follows the data, not the name, so an off-vocabulary clusterer is caught exactly like leiden, and a provably-predefined grouping vetoes an incidental clustering call.

Phase A's provenance is may-level, so it escalates rather than accuses: the verdict caps at needs_evidence; a hard blocker still needs init → confirm → audit on the offending step's data.
▶ claim attributionwhich reported sentence rests on the flawed test? pins the verdict

Each numeric claim in the report is backward-linked to the test that produced it — statically, no execution — and the verdict is pinned onto that sentence. "Cluster 3 had 58 markers at padj < 0.05" → needs_evidence, tied to the de-novo-cluster marker test. It certifies the claim's method — never that the reported number reproduces — and an ambiguous claim abstains rather than mis-attribute.
